# Book Architecture for Physical AI & Humanoid Robotics

## Site Structure

The book will be built using Docusaurus, a static site generator. The overall site structure will be as follows:

-   **Homepage**: `book-root/src/pages/index.tsx` - Introduction to the book and key modules.
-   **Blog**: `book-root/blog/` - For announcements, updates, and supplementary articles.
-   **Documentation (Modules)**: `book-root/docs/` - Core content, organized into modules. Each module will have its own subdirectory.

## Sidebar Hierarchy

The sidebar will be automatically generated by Docusaurus based on the `_category_.json` files within each module directory. The hierarchy will reflect the logical flow of the curriculum.

-   **Module 1**: ROS 2 Fundamentals
-   **Module 2**: Digital Twin & Simulation
-   **Module 3**: The AI-Robot Brain (NVIDIA Isaacâ„¢)
-   **Module 3B/Module 4**: VLA, Whisper & Humanoid Capstone (Current Module)

Each module will have an `index.md` as its landing page and subsequent `.md` or `.mdx` files for sections within the module.

## Content Modules

Each module will be a self-contained unit covering a specific topic in Physical AI and Humanoid Robotics. Content will be written in Docusaurus-ready Markdown.

-   **Module Content**: Detailed explanations, diagrams, step-by-step guides, examples.
-   **`_category_.json`**: Defines the module title and position in the sidebar.

## Documentation Flow

1.  **Specification (`specs/` )**: High-level requirements and user stories for each module/feature.
2.  **Plan (`specs/`)**: Detailed implementation plan, technical context, and project structure.
3.  **Tasks (`specs/`)**: Actionable, dependency-ordered tasks for implementation.
4.  **Content (`book-root/docs/`)**: Markdown files containing the actual book content.
5.  **Build & Deploy**: Docusaurus build process to generate the static website.

## Naming Conventions

-   **Module Directories**: `book-root/docs/00X-module-name/` (e.g., `004-module4-vla-capstone`)
-   **Content Files**: `kebab-case.md` (e.g., `whisper-to-action.md`)
-   **Category Files**: `_category_.json`

## Design Decisions Document

All significant design and content decisions (structure, theme, plugins, navigation choices) will be documented in a central location, potentially `book-root/docs/design-decisions.md` or within the relevant module's `research.md`.
